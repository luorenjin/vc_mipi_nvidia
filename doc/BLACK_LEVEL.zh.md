# 黑电平
通过以下设置向图像输出添加偏移量：
```
v4l2-ctl -d /dev/video0 -c black_level=<value>
```
其中该值的范围为 0 到 100000。此值被解释为千分之一百分比 (milli%) 或 m%。

黑电平值为 0 表示 0%，即不对像素输出添加任何偏移。
而值为 100000 则会向像素数据添加最大可能的偏移量。
0 到 100000 [m%] 之间的每一个值都将被解释为相对于传感器黑电平寄存器最大可能值的相对值。
例如：`black_level=5859` (5.859%) 在 8 位模式下将对应于 256 个可能值中的 15 个单位偏移。同样的比例关系也分别适用于 10 位或 12 位模式下的 60/1024 或 240/4096。这种相对属性的优点是，在不同位模式（Bit-modes）之间切换时，无需重新设置绝对值。

在驱动程序初始化时，将根据制造商的建议设置一个最佳默认值。遗憾的是，在用户手动修改该值之前，`v4l2-ctl` 不会反映出这个初始值。这意味着当驱动程序启动并根据给定的位模式设置了传感器默认值时，调用
```
v4l2-ctl -l
```
仍会显示 `black_level` 值为 0。目前对于这种行为尚无合适的解决方案，因为该控制项的初始值是保存在 `tegracam_ctrls.c` 中的。
